Ejercicio: Schema y Queries para Sistema de Usuarios
Dificultad: Media
Tiempo estimado: 45-60 minutos
Conceptos: DDL (CREATE TABLE), DML (INSERT, SELECT, UPDATE), JOINs, constraints, índices

Contexto
Vas a diseñar el esquema de base de datos para un sistema de usuarios y sus análisis biomecánicos. Este ejercicio simula el backend de tu SaaS real.

Objetivos de aprendizaje

Diseñar tablas con relaciones (1:N)
Aplicar constraints y validaciones a nivel DB
Escribir queries de consulta y filtrado
Entender índices para optimización
Pensar en integridad referencial
